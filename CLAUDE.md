# Starbucks Kaggle Project

## プロジェクトの目的
スタバをケーススタディにした空間データサイエンスのKaggle公開プロジェクト

## テーマ1の概要
全米30年の空間的増殖アニメーション+アニュアルレポートのNLPによる経営言語の時系列変化分析

## テーマ2の概要
マンハッタンにおけるスタバのドミナント戦略分析：
出店重複圧力の推定・需要代理変数ベースの立地適応度評価・歩行者流動/ビル属性による需要推定・店舗タイプ別立地分析(滞在型vsテイクアウト)・出店候補地ランキングとバックテスト

## データに関する制約条件
- 完全無料データのみ使用
- 再配布可能(CC0/CC-BY/ODbL/政府統計等)なもの中心
- 再配布不可のデータはnotebook内で取得手順のみ記載し、データ自体は同梱しない
- APIは最終検証のピンポイントのみ(従量課金に注意)
- スクレイピングは原則禁止(利用規約を必ず確認)

## datasetの公開方針
独立したdatasetとして公開。キー設計・欠損方針・更新履歴・ライセンスを明記する前提。

## 分析手法の概要
空間自己相関、OSMnxネットワーク分析、トピックモデリングまたはembeddingの時系列クラスタリング、需要代理変数による立地評価

## フォルダ構造
- data/raw/ : 生データ。絶対に上書き・削除しない
- data/interim/ : 中間処理済みデータ
- data/processed/ : 分析用最終データ
- data/external/ : 外部補助データ
- notebooks/ : テーマ別Notebook
- src/ : 再利用する関数モジュール
- reports/ : AIレポート出力・週次レビュー
- docs/ : 設計書・棚卸し表

## データ運用ルール
- data/raw/の中身は一切変更しない。加工はinterim/以降に出力する
- 既存ファイルを削除する処理は禁止(人間レビュー後のみ許可)
- 日付・バージョン付きで出力する(例: stores_cleaned_v2_20260312.csv)
- mainブランチに直コミット禁止。作業は必ずブランチを切る

## コード出力時の注意事項
- 空間分析でCRS(座標参照系)を変換する場合、変換前後のCRSを必ずコメントに明記すること
- 数値計算の結果は必ずサニティチェック(行数・値域・欠損数)をコードに含めること
- エラーを握り潰すtry/exceptは禁止。エラーが出たら報告すること
- メモリを大量消費する処理の前にデータサイズを確認するコードを入れること
